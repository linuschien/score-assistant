# US-05 成績登記 (Grade Recording)

## 背景 (Background)

教師針對特定成績項目（如出席、作業、課堂表現等），為班級內每位學生逐一登記分數或狀態，並可在事後修改。

---

## US-05-01：矩陣式成績登記 (Matrix Grade Recording)

**身份**：教師 (Teacher)

> **As a** 教師，  
> **I want to** 在一個二維矩陣介面中同時看見全班學生與所有成績項目，  
> **So that** 我可以橫向切換項目、縱向切換學生，快速且直覺地完成整體的成績錄入。

### 驗收條件 (Acceptance Criteria)

- **AC1**：系統提供「成績登記總表」介面，採二維矩陣結構：
  - **縱軸（列）**：顯示班級所有學生（學號、座號、姓名）。
  - **橫軸（欄）**：顯示該班級已建立的所有「成績項目」。
- **AC2**：每個單元格（Cell）代表特定學生在特定項目的成績紀錄，支援以下操作：
  - **分數輸入**：可直接輸入數值分數（支援小數點）。
  - **狀態選單**：若該項目為 `ATTENDANCE` 類型，提供下拉選單選擇 `出席`、`缺席`、`請假`，並自動換算為分數。
  - **附件觸發**：若該項目為 `ASSIGNMENT` 或 `REPORT` 類型，單元格內提供附件管理按鈕。
- **AC3**：分數規則：除 `CLASSROOM_PERFORMANCE` 類型外，分數不得低於 0；系統自動依項目設定的 `max_score` 進行上限檢核。
- **AC4**：教師可以隨時修改矩陣中的任一單元格，修改後即時（或點擊儲存後）更新該筆 `GradeRecord`。
- **AC5**：介面支援凍結標題列與學生資訊欄（學號、座號、姓名），確保在捲動查看大量評量項目時，仍能識別學生。
- **AC6**：`CLASSROOM_PERFORMANCE` 類型的單元格允許輸入負分，並在介面上有明顯視覺區分（如紅色字體）。

## US-05-02：修改已登記的學生分數

**身份**：教師 (Teacher)

> **As a** 教師，  
> **I want to** 修改特定學生在某成績項目的分數，  
> **So that** 我可以更正登記錯誤。

### 驗收條件 (Acceptance Criteria)

- **AC1**：教師可以重新進入成績項目的登記頁面，直接覆蓋修改任意學生的分數。
- **AC2**：修改後，系統記錄最後修改時間以供稽核。
- **AC3**：儲存成功後顯示成功提示。

## US-05-03：查看單一學生的所有成績

**身份**：教師 (Teacher)

> **As a** 教師，  
> **I want to** 查看特定學生在所有成績項目的分數，  
> **So that** 我可以綜合了解該學生的學習狀況。

### 驗收條件 (Acceptance Criteria)

- **AC1**：系統顯示選定學生的個人資訊（學號、座號、姓名、電子信箱）與成績一覽表，欄位包含：項目名稱、類型、日期、分數、滿分、權重。
- **AC2**：頁面底部顯示該學生的加權總分（詳見 US-07）。
